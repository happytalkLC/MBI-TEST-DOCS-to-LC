이전 상담사 연결 가능 여부¶

Endpoint¶

[POST] /v1/chat/isCounselor

Description¶

Header에 들어가는 인증 키 (HT-Client-Id, HT-Client-Secret)는 smurf 외부 메신저 API에서 key (HT-Client-Id)와 Token (HT-Client-Secret)를 발급 받으실 수 있습니다.

Request (Json)¶

Header¶

Field Name

Type

Required

Description

HT-Client-Id

String

Y

smurf에서 발급 받은 key

HT-Client-Secret

String

Y

smurf에서 발급 받은 token

Body¶

Field Name

Type (Length)

Required

Description

Default

uuid

String (20)

Y

고객사 고객 식별 정보 (최소 5자 ~ 최대 20자)

customer_id

String (20)

Y

고객사 고객 식별 정보

Field Name

Type

Description

code

Int

성공, 실패여부

String

상담사 이름

Example¶

Success¶

  • 이전 상담사가 존재 할경우

{
  "code": 0,
  "counselor": "exist",
  "counselor_code": 1,
  "counselor_name": "상담사"
}

  • 이전 상담사가 존재 하지 않을 경우

{
  "code": 0,
  "counselor": "not exist",
  "counselor_code": 0,
  "counselor_name": ""
}

Fail¶

  • 일치하는 값이 없음

{
    "code": 0,
    "counselor": "fail",
    "counselor_code": -1
}

  • 권한 , 페이지없음

{
  "code": -400,
  "message": "Forbidden"
}

  • Authorization 잘못 되었거나 만료된 정보 , 인증실패

{
  "code": -500,
  "message": "Authorization"
}

  • 전송 실패

{
  "code": -504,
  "message": "Contract"
}

  • 내부 시스템 오류

  "message": "System"

  "message": "EndRoom"
}

  • 상담방 생성 실패

  "message": "NotRoom"

  • 상담방 UUID가 다름

  • 현재 진행중인 상담방
